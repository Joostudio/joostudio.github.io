iOrderFoods

Improving a live ordering platform through practical UX thinking and cross-platform insights.

Overview

Many restaurants still rely on phone calls to manage pickup orders — leading to frequent mistakes, long wait times, and overworked staff. iOrderFoods was created to offer a digital solution, but early usability issues caused frustration for both customers and restaurant owners. To address this, I redesigned core areas across the ordering platform and notification app — including login, payment, menu clarity, real-time order alerts, and admin guidance — resulting in a more intuitive, self-serve experience.

My Role

User Research / Visual Design / Prototyping & Testing

  • Improved login & payment flows
  • Enhanced order visibility & admin access
  • Built in-app Help Center & onboarding shortcuts
  • Referenced UX patterns from Amazon, GAP, Nordstrom, iMessage

The Challenge : iOrderfoods Platform

1. Confusing Login Flow

"I clicked ‘Order’ but couldn’t figure out where to log in."


Problem: New users were confused by the login flow
Solution:Made the login screen appear immediately after clicking “Order,” reducing hesitation and entry friction


2. Payment Abandonment

"I thought I paid, but it didn’t go through…"


Problem: Customers often thought they had completed an order without finalizing payment
Insight: Online food ordering is a relatively new UX pattern, so I analyzed e-commerce leaders like Amazon and GAP.
I noticed that Amazon's CTA button is deliberately disruptive (yellow on dark navy) to prevent missed action. GAP and Rag & Bone also keep checkout buttons persistently visible.
Solution: I applied this strategy by keeping the “Pay” button clearly visible until payment was confirmed


3. Hard-to-Scan Menu Items

"Is this spicy? Vegan? I can't tell."


Problem: Dietary info (e.g., spicy, veggie) and featured items weren’t distinguishable
Design: Added visual icons and color tags to improve scannability and item clarity


4. Repetitive Support Calls

"How do I add a menu item again?"

Problem: Basic usage questions were generating frequent support calls
Design: Created a help center with documentation, embedded in the admin interface → Support calls dropped by ~90%


5. Missing Onboarding Guide

"How do I add a menu item again?"


Problem: Users couldn’t easily find onboarding materials or remember the help URL
Insight: I originally proposed linking the guide based on my past website management experience. When my manager asked for proof of similar industry use, I researched and found Nordstrom’s contextual help integration. I shared the example with the developer, and it led directly to implementation.
Solution:Embedded a permanent help link inside the admin menu


The Challenge : iOrderfoods Order Notification App

1. Missed New Orders

"We didn’t even notice a new order came in."

Problem: New or unread orders weren’t visually clear, leading to missed tickets
Insight: Inspired by iMessage, I used familiar visual cues (bold text, red dots, persistent alerts) to improve urgency and visibility
Solution: Highlighted unread orders using attention-grabbing UI patterns


2. Clunky Order Controls

"Can we change pickup time without going to the dashboard?"

Problem: Staff couldn’t easily modify delivery times or toggle online ordering
Solution: Added toggle switches and sliders for in-app control without needing admin panel access


3. Confusing Access to Admin Settings

"I’m not sure where the real settings are — is this the admin page?"

Problem: Users of the order notification app frequently confused it with the main eCommerce admin platform, making it difficult to access key settings
Insight: Drawing from Nordstrom’s contextual help model, I realized direct shortcuts reduce friction
Solution: Added a direct admin shortcut in the hamburger menu that opens a side panel with key management links for quick and intuitive access


Results

✅ Support call volume reduced by ~90%
✅ Checkout abandonment significantly decreased
✅ Increased clarity and confidence among restaurant owners and staff
✅ Reduced onboarding friction and training time
