iOffering

Designing a smoother, recurring donation experience for churches and their communities.

Overview

Many churches rely on consistent donations from members to support operations and outreach. However, the existing iOffering system only allowed one-time payments, requiring users to return each month and manually donate again. This friction led to inconsistent giving patterns and unnecessary workload for administrators.

To address this, I led the visual design and prototyping of a new recurring donation feature — enabling church members to set up monthly automated contributions with ease and control, while reducing repetitive tasks for staff.

My Role

Visual Design / Prototyping & Testing

  • Designed the recurring donation flow
  • Created clean, mobile-friendly UI
  • Collaborated with developers for smooth integration
  • Conducted usability testing and iteration

The Challenge

"I want to support my church monthly, but I always forget... it’d be nice if it just happened automatically."

Members wanted the ability to give regularly without remembering each time. Admins also struggled with manual tracking. However, the platform only supported one-time payments.

The Process

  • Discovered user and admin pain points through behavior analysis and internal feedback
  • Set UX goals: automation, transparency, mobile-first design
  • Designed a minimal step-by-step recurring donation flow
  • Tested and refined through user feedback

Design Solution

  • Simple toggle between one-time and monthly donations
  • Clear confirmation and editable preferences
  • Transparent billing and auto reminders
  • Integrated admin panel automation

Results

✅ Steady increase in recurring donations over time
✅ Significant reduction in donation tracking workload
✅ Higher satisfaction from both donors and administrators
✅ Improved competitive positioning with modern fintech-level UX
